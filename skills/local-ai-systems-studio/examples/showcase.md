# Showcase Examples

These examples show how the skill combines workflow design, deployment guidance, tool evaluation, and issue-to-plan conversion into one practical local AI operating flow.

## Example 1: LM Studio versus Ollama versus MLX

**Prompt**

`I use a Mac and want a local Qwen workflow for document summaries. Should I use LM Studio, Ollama, MLX, or a GGUF setup?`

**A strong output should include**

- hardware-aware reasoning
- clear tradeoffs between GUI convenience and scriptability
- one recommended primary stack
- a short setup and validation path

**Why this showcases the skill**

It demonstrates that the skill can make practical local stack decisions instead of giving generic tool lists.

## Example 2: Local AI Workflow Design

**Prompt**

`Help me build a local workflow that reads PDFs, extracts action items, and creates a reusable summary format.`

**A strong output should include**

- stage-by-stage workflow design
- model work separated from tool work
- realistic guidance on what to automate first
- a simple validation loop

**Why this showcases the skill**

It shows the skill can bridge model choice, workflow structure, and implementation planning.

## Example 3: Local Issue Triage

**Prompt**

`My local AI workflow is slow and the outputs are inconsistent. Help me decide what to fix first.`

**A strong output should include**

- bottleneck framing
- comparison of likely root causes
- prioritized actions
- a concrete plan rather than vague troubleshooting advice

**Why this showcases the skill**

It proves the skill is useful not just for initial setup, but for improving a real local AI system over time.
