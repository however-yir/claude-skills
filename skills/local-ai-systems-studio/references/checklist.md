# Local AI Systems Checklist

## Frame the Real Task

- What repeated job should the system perform?
- What input format starts the flow?
- What output format must be stable?
- What matters most: speed, quality, privacy, or setup simplicity?

## Compare the Options

- Does the hardware fit MLX, GGUF, or a serving stack better?
- Does the user need GUI convenience or scriptable APIs?
- Is the system mostly prompt-only, tool-using, or retrieval-heavy?
- What is the maintenance cost of each option?

## Choose the Simplest Viable Stack

- Pick one primary recommendation.
- Keep one fallback path.
- Avoid multi-tool complexity unless it solves a real problem.

## Turn It Into a Plan

- Define milestones.
- Define validation checkpoints.
- List likely bottlenecks.
- Choose the next smallest useful action.
