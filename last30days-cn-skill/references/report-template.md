# Report Template

Return the report in Chinese and keep the section order below.

You may include a one-line preface before section 1 for:

- 时间窗口
- 实际覆盖到的平台
- 缺失的平台

Do not add a long methodology section unless the user asks for it.

## 1. Top Trends

Return `5-10` trend items.

For each trend, use this structure:

```markdown
#### {趋势标题}
- 摘要：2-3 句话，说明发生了什么
- 趋势形态：短期 hype / 持续趋势
- 为什么重要：一句到两句，讲清业务、产品、内容或舆论意义
- 趋势平台：X / Weibo / Xiaohongshu / Douyin
- 代表性案例：
- X：平台内代表性帖子或讨论角度，附一句互动或传播描述
- Weibo：平台内代表性帖子、热搜或超话角度，附一句互动或传播描述
- Xiaohongshu：代表性笔记或内容套路，附一句互动描述
- Douyin：代表性视频、话题或挑战，附一句传播描述
```

Rules:

- examples should be short descriptions, not raw links
- each trend should have at least 2-3 total examples across all covered platforms
- if a platform has no evidence for that trend, omit that platform line for the trend

## 2. Cross-Platform Insights

Include:

- which topics appear on multiple platforms
- how the narrative differs between China-local platforms and X
- where sentiment, framing, or creator behavior diverges

Keep this section analytical and comparative.

## 3. Emerging Signals

List weak but fast-growing signals worth watching.

For each signal, briefly state:

- what the signal is
- why it might accelerate
- which platform shows the earliest traction

## 4. Key Takeaways

Return `3-5` high-level takeaways about the current internet landscape.

Good takeaways usually describe:

- what is getting attention
- what kind of content format is winning
- whether China and global narratives are converging or diverging
- what feels durable vs temporary

## Style Rules

- concise
- evidence-driven
- internet-native wording is fine
- avoid generic statements like “users are increasingly interested”
- prefer concrete descriptions like “AI agent tooling moved from abstract discussion to workflow comparison and cost-performance debates”
